# /commit

このスキルを実行する前に、必ず `.github/commit-message-instructions.md` を読み取り、
そこに定義されたフォーマットとルールに従ってコミットメッセージを生成してください。

## 実行プロセス

1. **規約の読み取り**
   - `.github/commit-message-instructions.md` を読み込み、フォーマット・型・記述ガイドラインを把握する。

2. **差分の網羅的検証**
   - `git status` および `git diff` を実行し、全ての変更箇所を確認する。
   - 変更内容を論理的なまとまり（Atomic Commit）として整理する。

3. **ブランチ整合性のチェック（重要）**
   - 現在の作業ブランチ名（例: `feat/xxx`, `fix/yyy`）と実際の差分内容を照らし合わせる。
   - **不適切な差分の検知**:
     - `feat` ブランチなのにバグ修正（`fix`）が混入している。
     - 本来のタスクとは無関係なリファクタリングが含まれている。
   - **アクション**: 不適切な差分が存在する場合はコミットを中断し、別ブランチでの作業や `git stash` を用いた整理をユーザーに推奨する。

4. **メッセージの生成**
   - `.github/commit-message-instructions.md` の「基本フォーマット」「型の定義」「記述ガイドライン」に従ってメッセージを生成する。
   - 大規模な差分の場合は同ファイルの「詳細の構造化」ルールに従い、本文（Body）を付与する。

5. **実行と報告**
   - 整合性が確認された差分のみを `git add` し、コミットを実行する。
   - 実行後、ハッシュ値・適用したメッセージ・整理した差分の概要を報告する。

## 制約

- 整合性チェックをパスしない限り、無理にコミットを進めてはならない。
- `git push` はユーザーからの別途指示を待つこと。
- メッセージは必ず日本語で出力すること（`.github/commit-message-instructions.md` の言語ルールに準拠）。
